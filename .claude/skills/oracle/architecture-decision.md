# Skill: Architecture Decision Record

## Framework
1. **Context**: Problem, constraints, success criteria
2. **Options**: ≥2 viable approaches with pros/cons/risk/effort
3. **Analysis**: Score each option across correctness, performance, simplicity, maintainability, extensibility, risk
4. **Recommendation**: One option, confidence level, deciding factor, caveats
5. **Consequences**: What becomes easier/harder

## Anti-Patterns
- ❌ Only one option considered
- ❌ Over-engineering for hypothetical futures
- ❌ Novelty over reliability
- ❌ Ignoring existing patterns
- ❌ Unsubstantiated claims
